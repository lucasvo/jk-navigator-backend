Using the extension

Using the extension is fairly simple. Whenever you are on a pages where j/k is supported, you can use the following shortcuts to navigate through the list of links:

  • j Next result
  • k Previous result
  • i Back to first position
  • o or Return Open result
  • ⌘ + Return Open result in new tab/window
  • / Focus search box

Once you've opened the first link using o you can also cycle through the links without returning to the search page but just by pressing j or k.

Adding custom sites

Adding custom sites is easy but does require a knowing how to work with jQuery selectors.

To add a site you need to do two things: Regex:Define a regex to match for sites you want to enable JK on and create a simple JSON object defining the selectors to use for it. The second part is defining the JSON object containing the selectors. It's easiest if you look at some of the built in sites for pointers on how to write them.

selectors: css selector(s) for the links that we want to navigate.
paginator_selector_next: css selector for the link that opens next page of results
paginator_selector_prev: css selector for the link that opens next page of results
allowSubdomains: whether or not subdomains should trigger the jk navigator (www is not treated as subdomain here and will always work). docs.google.com for example has its own jk functionality and should not be broken through the one on google.com
search_selector: css selector for the search input field
liveUpdateElement: for pages with ajax, such as Google and Facebook, that load search results dynamically, we need to know the parent liveUpdate Element so that we can load jk at the right time when results are here.
infiniteScroll: set to True if we have a page like Facebook or Quora that can dynamically load new links into the link list.

Privacy

JK Navigator does nothing evil. There is no tracking and at no point in time is any behavior transmitted to jknavigator.org. The JK Navigator is available open source under an BSD License, you can go check out the source on github.com to find out more.{# TODO LINK #}

The only time we are ever transmitting any user data to jknavigator.org is when you add a custom site and chooose to submit the information to us. All this data is used for, is to determine which sites we should add next. The JK Navigator periodcally queries our server to update the default site configurations.
